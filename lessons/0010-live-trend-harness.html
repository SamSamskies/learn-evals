Anthropic calls this the harness: the machinery that hands the task to the model and collects the transcript. If the harness builds a different prompt than production, you are baselining a product that does not exist.

What transfers, what doesn't

Transfers

The gate (LOCALLAB_LIVE_EVAL=1 + skipIf), the afterAll pass-rate summary, and the raw-answer dump with begin/end markers your compare-live-evals skill parses.

Changes

The context: buildChatSystemPrompt({ type: "trend", series, panels }) instead of the panel shape. Each fixture already carries its own series and panels.

Watch out

scripts/live-eval.mjs hardcodes the panel test file as a CLI argument. Left alone, your new trend file will silently never run live.

The gating is layered, and it is worth seeing why. The default vitest.config.ts excludes server/**/*.live.eval.test.ts, so naming the file correctly keeps npm test offline-safe with zero effort. The describe.skipIf(!LIVE_EVAL_ENABLED) inside the file is the second lock: even if someone points vitest at the file directly, nothing burns inference time without the flag.

Practice: wire it, gate it, baseline it

In LocalLab:

  1. Create server/trendChat.live.eval.test.ts mirroring the panel live suite: same env gate and timeout handling, but build context with buildChatSystemPrompt({ type: "trend", series: level1Case.series, panels: level1Case.panels }), call generateChatReply, score with evaluateTrendChatLevel1(answer, level1Case) over TREND_CHAT_LEVEL1_CASES. Keep the afterAll pass-rate log and the raw answer begin/end markers.
  2. Update scripts/live-eval.mjs so trend runs: add a --suite panel|trend|all flag (default all) that picks which file paths to pass to vitest — the live config's glob already includes both.
  3. Baseline on gemma4:26b: npm run test:live-eval -- --suite trend --model gemma4:26b. Record pass rate and wall time per case.
  4. Triage every fail with the grader-triage ritual before changing anything: true fail, grader false positive, or flake. Grader FPs earn a canned fixture; flakes earn more trials — never a diluted assert.
  5. Confirm plain npm test is untouched and offline-green.

If the leading case trips shared diagnosis bans on softened language, that is the same hotspot Lesson 7 hardened for panels — compare against isSoftenedOrMetaHypothyroidClaim before writing anything new.
