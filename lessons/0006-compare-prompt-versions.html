Change the prompt, not the model

Hold medgemma1.5:latest and Level 1 fixed. Aim the variant at a known fail—then decide with evidence.

The tangible win

By the end, you will run Level 1 twice on medgemma1.5:latest—baseline guidance vs one deliberate prompt variant—fill two rows on the comparison card (label the prompt version), and decide whether the variant helped that failure without making Level 1 worse.

fair prompt compare = same cases + same graders + same model + one prompt change

Hamel’s FAQ warns that automated prompt optimizers hill-climb known metrics but do not discover new failures. Your Level 1 suite is a known metric bank: use it to verify a human-authored prompt edit aimed at a failure you care about—not to blindly rewrite CHAT_GUIDANCE.

Edit for a reason, then measure

Start from the known medgemma miss: over-confident hypothyroid language on elevated-tsh-leading. Tighten one guidance clause, re-run the suite on the same model. If the fail does not move, that is useful too—it may be a capability gap, not a wording gap. Do not delete the assertion to manufacture green.

Anthropic treats a static task bank as how you adopt changes quickly. Your bank already exists; this lesson is the prompt half of the ritual you used for models.

Practice: two prompt rows on medgemma

In LocalLab, keep fixtures, asserts, and medgemma1.5:latest fixed:

  1. Aim at the known fail: elevated-tsh-leading: no-hypothyroid-diagnosis (or another medgemma miss you reproduce). One sentence describing the failure is enough.
  2. Duplicate or branch CHAT_GUIDANCE / prompt construction so you can run baseline vs variant without editing asserts. Smallest hook wins (env flag, test-only override, or a temporary second export).
  3. Run npm run test:live-eval -- --model medgemma1.5:latest once per prompt version. Record pass rate, failing ids, and wall-clock on the comparison card (use the Prompt version field).
  4. Decision in one sentence: keep baseline, adopt variant (then spot-check once on gemma4:26b before shipping), or treat the miss as capability and stop prompt-chasing.

Do not change the model mid-bake-off. Do not loosen asserts to make the variant green. Do not replace gemma4 as the product default just because medgemma is a better teaching workbench.
