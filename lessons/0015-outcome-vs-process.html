Grade the path when luck lies

Final-text Level 1 is an outcome check. When work becomes multi-step, a lucky reply can hide a bad trajectory—learn to grade both without freezing one brittle script.

The tangible win

By the end, you will ship a thin offline outcome + process harness in LocalLab: one synthetic multi-step transcript shape, outcome asserts on the final reply, process asserts on tool use (required / forbidden—not exact order), and canned Vitest covering a good run, a lucky-bad-path run, and a bad-outcome run.

What you just proved: Lesson 14 promoted hdl-stable from a banked invent-direction pattern. Panel and trend Level 1 still grade the final reply string. That is correct for single-turn chat. This lesson is the bridge before tools or multi-turn agents exist—so you already know what to measure when they do.
outcome = final state · process = steps that produced it

Anthropic separates the transcript (full trial record) from the outcome (final environment state). A flight agent can say “booked” while no reservation exists—or the reverse. For agents, Hamel pairs outcome metrics with process metrics; process failures are often easier to debug because they are more deterministic. Early-stage misses cascade.

Three failure shapes

Bad outcome

Tools may look fine; the final reply invents a rise on flat HDL. Your current Level 1 already catches this.

Lucky path

Final reply looks grounded, but the agent never called lookup—it hallucinated numbers that happened to match. Outcome-only passes; process must fail.

Brittle script

Requiring exact tool order A→B→C punishes valid alternate routes. Anthropic prefers grading what was produced over one scripted path.

LocalLab-shaped toy agent

Imagine a tiny trend helper (not production yet) with one tool: getTrendSeries(marker). User asks “Is my HDL rising?” on a flat 55→55 series. A good trajectory calls the tool, then answers with stable language. A lucky trajectory skips the tool but still says “stable at 55.” A bad trajectory calls the tool and still invents a rise.

outcome: stable language + cite 55 · process: getTrendSeries called · forbid: writeDb

Practice: thin transcript harness

In LocalLab (offline only—no live agent loop required):

  1. Add a small module (e.g. under server/evals/) with a typed transcript shape: userMessage, toolCalls[] (name + args), finalReply. Seed one synthetic flat-HDL scenario.
  2. Write outcome graders on finalReply (reuse Level 1 instincts: cite 55, state stable, no false rise). Write process graders on toolCalls: getTrendSeries present; writeDb (or any write tool) absent. Do not require a single exact call order beyond “lookup before answer” if you only have one call.
  3. Ship three canned transcripts + Vitest: (A) good outcome + good process, (B) good outcome + skipped tool (must fail process), (C) called tool + invented rise (must fail outcome). Offline green. Do not wire Ollama or change the ship gate.

Keep fixtures synthetic. Do not replace panel/trend Level 1 with this toy. Do not add a second inference provider.
