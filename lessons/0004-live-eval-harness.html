harness = run task → record answer → grade → aggregate

Anthropic defines an evaluation harness as the infrastructure that runs tasks end-to-end, records the work, applies graders, and aggregates results. Your canned tests proved the graders. A live harness proves whether a model + prompt path still satisfies them.

Production parity beats a toy caller

Anthropic’s hard rule: the agent under eval should behave roughly like the agent in production. In LocalLab that means reuse what the product already uses—not a one-off prompt string paste.

Gate live runs; keep canned always green

Live Ollama calls are slow, flaky without a daemon, and unsuitable as an unconditional CI gate yet. Keep your canned assertion tests always-on. Put live scoring behind an explicit opt-in.

Pattern:

  1. Keep existing canned pass/fail tests unchanged (they guard grader correctness).
  2. Add a second describe (e.g. “panel chat Level 1 live”) that skips unless process.env.LOCALLAB_LIVE_EVAL === "1".
  3. Read model from process.env.LOCALLAB_EVAL_MODEL (or your usual Ollama default) and fail loudly if unset when live mode is on.
  4. Raise Vitest timeout for that block (LLM replies often need tens of seconds).
  5. On failure, log the full answer text—Anthropic’s “read the transcripts” rule starts here.

One trial is enough to start; expect noise

Models vary run to run. Anthropic separates trials from tasks for that reason. For this lesson, one trial per case is enough to stand up the loop. If a case flickers, treat that as signal: either harden the assertion, fix the prompt, or plan multi-trial metrics later—not an excuse to abandon Level 1.

Aggregate simply for now: cases with every assertion passing / cases run, and list which assertion ids failed. Pass rate without failing assertion ids is a vanity number.

Practice: wire the live harness

In LocalLab, add a gated live block that loops PANEL_CHAT_LEVEL1_CASES:

  1. buildChatSystemPrompt({ type: "panel", panel: level1Case.panel })
  2. await generateChatReply(…, [], level1Case.userMessage, noopToken, model)
  3. Expect evaluatePanelChatLevel1(answer, level1Case).every((r) => r.pass)
  4. On fail, console log case id, failing assertion ids, and the raw answer

Run with your env flag + model set against one Ollama model you already use for chat. Stay on panel chat only. Do not add a model judge yet.
