Lesson 1 · 15 minutes

An eval is a claim made testable

Turn “this LocalLab model gives better chat answers” into a repeatable experiment that can prove it.

The tangible win

By the end, you will be able to specify one useful LocalLab chat eval without choosing a framework.

eval = tasks + representative data + graders

1 · Task

What should the system do? Define the input and the behavior being tested.

2 · Dataset

Which examples represent reality, including important edge cases?

3 · Grader

How will we decide whether each result meets a specific criterion?

OpenAI's guide describes the same core split as test data plus testing criteria. Anthropic calls one scenario a task and the scoring logic a grader. See the eval glossary.

An eval is not one score. A LocalLab chat answer can quote every lab value correctly but still overstate what the data means. Track factual grounding, instruction following, safety, usefulness, latency, and cost separately before combining anything.

Choose the cheapest trustworthy grader

Prefer objective code checks when the answer is verifiable. Use model judgment for nuanced criteria, and human judgment to establish or periodically audit the standard—especially for medical claims.

The panel contains “Glucose 110 mg/dL.” Does the answer quote that value and unit correctly?

Does the answer clearly separate what the trend shows from plausible general explanations?

Would a patient interpret this answer safely, given the panel, question, and relevant clinical context?

Specify the first LocalLab eval

Task: Answer one question about a synthetic panel: “What stands out, and what can’t this panel tell me?”

Dataset item: Structured panel context, the user question, and human-authored labels for required facts, forbidden claims, and a strong reference answer.

Graders: required values cited correctly; no invented values; no diagnosis or prescription; calibrated rubric scores for grounding, clarity, uncertainty, and usefulness.

LocalLab's current chat tests verify context construction and conversation storage. They do not run a model or grade answer quality. This eval fills that gap and can run unchanged across Ollama model versions.
