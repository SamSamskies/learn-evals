Three different reds

A red assertion id is not one failure mode. For LocalLab’s TSH safety check, you already saw all three:

True fail

The model answers the leading question with a diagnosis. Product change is fair game.

False positive

Regex hits refusal or meta text (“diagnose hypothyroidism” inside a clinician deferral; “you have hypothyroidism” as a bad example in a thinking trace).

Flake

Same model + prompt + cases + graders; pass↔fail across runs. One card cannot gate ship.

Practice: triage, harden, then measure agreement

In LocalLab, keep production on the default prompt. Work the grader:

  1. Collect at least two transcripts that hit elevated-tsh-leading: no-hypothyroid-diagnosis (from today’s bake-offs or a fresh live run). Label each using the triage sheet.
  2. Harden no-hypothyroid-diagnosis so clinician-deferral / example-to-avoid wording passes while a clear absolute diagnosis still fails. Add a canned unit test (or fixture reply) for at least one known false positive.
  3. Freeze one config (e.g. gemma4:26b + default prompt). Run live Level 1 three times. Record per-run pass rate and whether no-hypothyroid-diagnosis agreed across all three (this is the practical start of pass^k-style consistency).
  4. One-sentence verdict: is Level 1 calm enough to re-run default vs stricter-no-diagnose as a ship decision? If not, what FP or flake remains?

Do not promote the prompt variant mid-lesson unless your three-trial agreement and hardened check clearly support it. Do not delete the TSH safety assert.
