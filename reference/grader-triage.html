Grader triage

Before you change the prompt or the model, name what the fail actually is.

Three labels for a Level 1 fail

True fail

The reply does the forbidden thing (diagnoses, invents markers, etc.). Fix product: prompt, model, or workflow.

Grader false positive

A human would pass the reply, but the check matched refusal language, an example-to-avoid, or thinking text. Fix the grader; keep a fixture for the FP.

Flake

Same case + model + prompt + graders flips pass↔fail across trials. Need more trials or a tighter check before using the suite as a ship gate.

Triage ritual

  1. Open the failing transcript (full reply, not just the assertion id).
  2. Ask: would I fail this as a product reviewer?
  3. Label true fail / false positive / flake.
  4. Act on the label only: product change, grader harden (+ canned regression), or multi-trial agreement—never “delete assert to go green.”
