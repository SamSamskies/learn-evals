Reference · Suite growth

Online → offline

Grow Level 1 from failure patterns you actually saw—not from imagined coverage checklists.

The loop

observe → categorize → synthesize fixture → assert → re-run (canned, then live)
  1. Observe — read transcripts from live evals, bake-offs, ship gates, or (later) privacy-reviewed product traces.
  2. Categorize — name the failure mode in one short phrase (e.g. “invents direction on flat series”). Group repeats; ignore one-offs until they recur.
  3. Synthesize — build a fake panel/trend fixture that triggers that mode. Never paste raw medical production content into the repo.
  4. Assert — add shared or fixture-specific Level 1 checks that fail a canned “bad” reply and pass a canned “good” one.
  5. Re-run — offline canned first; live when the case joins a suite used for ship decisions. Re-clear pass^k before shipping if the case is in the gate.

Error-analysis row (copy)

source id · suite (panel|trend) · primary issue · category · promote? (y/n) · notes

Start tiny: a few rows beat a metrics dashboard. Patterns drive tests; generic scores do not.

LocalLab privacy bar

  • Fixtures stay synthetic or deliberately de-identified.
  • Promote the pattern, not the patient’s numbers or narrative.
  • Real production logs need an explicit privacy review before any promotion (see RESOURCES.md Gaps).

What not to do

  • Invent a dozen cases from imagination with no observed pressure.
  • Dilute asserts so an observed true fail goes green.
  • Blend panel and trend when reporting the new case’s suite.
  • Skip canned Vitest before live Ollama.
