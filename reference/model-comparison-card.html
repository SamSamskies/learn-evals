Model comparison card

Hold everything fixed except the model. Record quality first, then wall-clock.

Constants (never change mid-compare)

  • Golden cases — same PANEL_CHAT_LEVEL1_CASES
  • Graders — same Level 1 assertions
  • Harness — same buildChatSystemPrompt + generateChatReply
  • Trial budget — same trials per case (start with 1)

Per-model row

Model: _______________

Pass rate: ___ / 3 cases

Failing assertion ids: _______________

Suite wall-clock: ____ s

Decision note (1 sentence): _______________

For LocalLab Ollama runs, monetary cost is roughly free; still note latency so later Routstr candidates can be scored on the same card with cost added.

Decision order

quality (pass rate + failing ids) → latency → cost

A model that fails grounding or safety asserts is slower-looking but still the wrong default. Prefer a slightly slower model that clears Level 1 over a fast model that invents markers.
