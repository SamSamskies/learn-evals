Eval glossary

The small vocabulary needed to discuss an evaluation system precisely.

Task (test case)

One input or scenario with defined success criteria.

Dataset

A collection of representative tasks, often including human-verified expected outputs or labels.

Golden dataset

A curated, trusted dataset used as a stable comparison point. “Golden” describes its quality and role, not its size.

Trial

One attempt to run a system on one task. Nondeterministic systems may need multiple trials per task.

Grader

Logic or judgment that scores an output or outcome against a criterion. A task can use several graders.

Code-based grader

A deterministic check such as exact matching, schema validation, numeric comparison, or a unit test.

Level 1 assertion

A fast, cheap code-based check on model output—Hamel Husain’s “LLM unit test.” It catches obvious regressions; it does not replace model or human judgment for nuanced quality.

Synthetic fixture

A deliberately fake, representative task input used for evals. For LocalLab chat: structured panel or trend context, a user question, and labels for required or forbidden content.

Model-based grader

An LLM applying a rubric to nuanced output. It scales subjective review but must be calibrated against humans.

Human grader

A person—often a domain expert—judging quality or correctness. Expensive, but the reference for subjective and high-stakes criteria.

Eval harness

The infrastructure that runs tasks end-to-end, records outputs (and optionally transcripts), applies graders, and aggregates results. Prefer production-parity paths so you evaluate the real product loop.

Eval suite

A collection of related tasks that share a broad goal—for LocalLab, “Level 1 panel-chat safety and grounding” is one suite.

Transcript

The full record of a trial (prompt, messages, model answer). Read failing transcripts before changing prompts or graders; pass rate alone hides false graders.

Regression eval

A suite asking whether the system still handles behavior it previously handled reliably.

Capability eval

A suite asking what the system can do well—often starting at a lower pass rate so the team has a hill to climb. After high pass rates, capability tasks can graduate into the regression suite.

Baseline

A recorded result on a fixed suite (pass rate, failing assertion ids, latency, later cost) used as the comparison point when changing model, prompt, or harness.

Model comparison

Running the same tasks, graders, and prompt while varying only the model. Quality first; then latency. For LocalLab: same Level 1 golden set across Ollama candidates.

Prompt comparison

Running the same tasks, graders, and model while varying only the system prompt (or prompt builder). Ship a prompt change only if Level 1 does not regress on failures you refuse to accept.

Acceptance gate

A fixed suite that must pass (or have explicitly accepted failures) before shipping a model default, prompt change, or workflow switch. For LocalLab chat: panel Level 1 and trend Level 1 both clear pass^k (start k = 3) on the same exact model tag—report each suite separately (never a blended 6/6). See the ship-gate sheet.

Trend chat task

An eval task whose context is a marker series across visits (plus underlying panels), not a single report. New pressures: direction/change, multi-point citation, invent-forbids scoped to those visits. Start offline (canned Level 1) before live Ollama scoring.

Grader false positive

The check fails a reply a careful human would pass—often because a regex matched refusal language, an example-to-avoid, or thinking text. Fix the grader; keep a fixture so the FP cannot return.

Flake

The same task + system + graders flips pass↔fail across trials. Treat as unstable signal: read transcripts, harden the check, or raise the trial budget before using the suite as a ship gate.

pass@k / pass^k

Multi-trial metrics from Anthropic’s taxonomy. pass@k: at least one of k trials succeeds. pass^k: all k trials succeed (consistency). LocalLab chat ship decisions care more about consistency than a lucky single green run.
