Eval glossary

The small vocabulary needed to discuss an evaluation system precisely.

Task (test case)

One input or scenario with defined success criteria.

Dataset

A collection of representative tasks, often including human-verified expected outputs or labels.

Golden dataset

A curated, trusted dataset used as a stable comparison point. “Golden” describes its quality and role, not its size.

Trial

One attempt to run a system on one task. Nondeterministic systems may need multiple trials per task.

Grader

Logic or judgment that scores an output or outcome against a criterion. A task can use several graders.

Code-based grader

A deterministic check such as exact matching, schema validation, numeric comparison, or a unit test.

Level 1 assertion

A fast, cheap code-based check on model output—Hamel Husain’s “LLM unit test.” It catches obvious regressions; it does not replace model or human judgment for nuanced quality.

Synthetic fixture

A deliberately fake, representative task input used for evals. For LocalLab chat: structured panel or trend context, a user question, and labels for required or forbidden content.

Model-based grader

An LLM applying a rubric to nuanced output. It scales subjective review but must be calibrated against humans.

Human grader

A person—often a domain expert—judging quality or correctness. Expensive, but the reference for subjective and high-stakes criteria.

Eval harness

The infrastructure that runs tasks end-to-end, records outputs (and optionally transcripts), applies graders, and aggregates results. Prefer production-parity paths so you evaluate the real product loop.

Eval suite

A collection of related tasks that share a broad goal—for LocalLab, “Level 1 panel-chat safety and grounding” is one suite.

Transcript

The full record of a trial (prompt, messages, model answer). Read failing transcripts before changing prompts or graders; pass rate alone hides false graders.

Regression eval

A suite asking whether the system still handles behavior it previously handled reliably.

Sources: OpenAI eval guide and Anthropic's eval taxonomy.
