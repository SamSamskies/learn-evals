# learn-evals

Personal learning track for becoming an AI Product Engineer who can change prompts, models, and AI workflows with evidence rather than intuition.

The proving ground is LocalLab's panel and trend chats (local Ollama). This repo holds the course materials — lessons, reference sheets, and learning records — not the LocalLab product code itself.

## Start here

1. Read [`MISSION.md`](./MISSION.md) for the why and success criteria.
2. Open lessons in a browser from [`lessons/`](./lessons/) (start with `0001-anatomy-of-an-eval.html`).
3. Use the teach skill / agent in this workspace to continue the track.

## Layout

| Path | Role |
| --- | --- |
| [`MISSION.md`](./MISSION.md) | Why this track exists and what “done” looks like |
| [`lessons/`](./lessons/) | Self-contained HTML lessons (primary teaching unit) |
| [`reference/`](./reference/) | Printable cheat sheets and glossaries |
| [`learning-records/`](./learning-records/) | Decision-style notes on non-obvious lessons |
| [`RESOURCES.md`](./RESOURCES.md) | Curated primary sources |
| [`NOTES.md`](./NOTES.md) | Working scratchpad (progress, preferences, next steps) |
| [`assets/`](./assets/) | Shared lesson styles and components |

## Constraints (high level)

- Lessons stay short (about 15–30 minutes).
- Prefer TypeScript and the existing LocalLab stack.
- Synthetic or deliberately de-identified medical fixtures only.
- Local Ollama only for this track.

See [`MISSION.md`](./MISSION.md) for full scope and out-of-scope items.
